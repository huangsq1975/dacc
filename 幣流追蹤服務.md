# 幣流追蹤服務

> **文件版本：** v1.0
> **建立日期：** 2026-04-28
> **適用範圍：** crypto-event-subscriber、aegisvault-transaction、crypto-event-ws、transaction-listener

---

## 一、概述

幣流追蹤（Currency Flow Tracking）由四個協作服務組成，負責監聽鏈上交易事件、解析幣流語義、即時推播狀態，並支援私有鏈的輪詢補充。

| 服務 | 語言 | 職責 |
|---|---|---|
| `crypto-event-subscriber` | Golang | 訂閱公鏈地址事件，發布至 Kafka |
| `aegisvault-transaction` | Golang | 消費 Kafka 事件，透過 PSL API 解析幣流語義並持久化 |
| `crypto-event-ws` | Golang | 消費 Kafka，透過 WebSocket 即時推播至前端 |
| `transaction-listener` | Golang | RPC 輪詢私有鏈，補充無事件訂閱的鏈路 |

---

## 二、資料流向

```
鏈上事件（公鏈）
      │
      ▼
crypto-event-subscriber（訂閱採集）
  ├── CryptoAPIs Provider（EVM / UTXO 公鏈）
  └── AegisAPIs Provider（內部私有鏈）
      │
      ▼  Kafka Topic
      │
      ├──► aegisvault-transaction（語義解析）
      │         │
      │         ▼
      │    PostgreSQL（ws_raw_transaction）
      │
      └──► crypto-event-ws（WebSocket 廣播）
                │
                ▼
           aegis-vault-app（前端即時更新）


鏈上事件（私有鏈，無事件訂閱）
      │
      ▼
transaction-listener（RPC 輪詢）
      │  Webhook
      ▼
aegis-vault-api（下游通知）
```

---

## 三、服務說明

### 3.1 crypto-event-subscriber（幣流事件採集）

**職責：** 訂閱鏈上地址的交易事件，依網路選擇不同資料提供商，將事件發布至 Kafka Topic 供下游消費。

**設定檔：** `config/config.yml`（從 `config/config.yml.bak` 複製）

**啟動：**
```bash
cp config/config.yml.bak config/config.yml
# 編輯 config.yml，填入 db / kafka / cryptoapi 設定
go build -tags dev ./...
```

#### 模組說明

| 套件 | 職責 |
|---|---|
| `pkg/subscription` | 訂閱 / 取消訂閱地址的交易事件；呼叫 provider 層取得幣流數據，透過 Kafka producer 發布結果 |
| `pkg/provider` | 提供商抽象層，依 config 中的 networks 設定選擇對應提供商 |
| `pkg/provider/cryptoapis` | 對接 CryptoAPIs 外部服務，實作 `ListTransactions`、`tx_subscription`、`tx_unsubscription` |
| `pkg/provider/aegisapis` | 對接內部 Aegis API，提供私有鏈的幣流訂閱能力 |
| `pkg/listtxlog` | 記錄每次幣流查詢的操作 log，寫入資料庫留存稽核 |
| `pkg/kafkaworker` | Kafka producer / consumer，負責發布事件及消費訂閱指令 |
| `pkg/callback` | 處理 CryptoAPIs 的 callback 回調，更新 `api_callback` 記錄 |
| `pkg/consumer` | Kafka consumer，消費訂閱 / 取消訂閱指令並觸發對應流程 |

#### 主要 API 操作

| 操作 | 說明 |
|---|---|
| `ListTransactions` | 查詢地址的歷史幣流記錄 |
| `tx_subscription` | 訂閱指定地址的鏈上事件 |
| `tx_unsubscription` | 取消訂閱指定地址 |
| `listtxlog.Create` | 寫入查詢 log 至資料庫 |

#### 設定重點（`config.yml.bak`）

```yaml
service.name: "crypto-event-subscriber"

http:
  listen:
    addr: "127.0.0.1"
    port: 8080

db:
  host: "localhost"
  port: 3306
  name: "crypto_event_db"

kafka:
  topic.prefix: "DEV_"
  broker.urls: "localhost:9092"
  consumer.group: "crypto-event-group"

cryptoapi:
  base.url: "https://rest.cryptoapis.io"
  apikey: "****"
  callback:
    url: "https://****.aegiscustody.com/callback/cryptoapis"
    key: "****"
```

#### 資料庫 Schema

| 資料表 | 說明 |
|---|---|
| `api_subscription` | API 訂閱記錄（network + address 唯一） |
| `ws_subscription` | WebSocket 訂閱記錄（含 client_id、address_type、is_active） |
| `api_callback` | CryptoAPIs callback 回調記錄（含 tx_hash、confirmations） |
| `ws_callback` | WebSocket callback 狀態（pushed / received） |
| `list_transaction_log` | 幣流查詢操作稽核 log |

### 3.2 aegisvault-transaction（幣流語義解析）

**職責：** 消費 Kafka 的原始交易事件，透過 Pine Street Labs（PSL）API 解析交易語義（付款方、收款方、金額、手續費），並維護多鏈地址的幣流狀態。

**設定檔：** `config/config.yml`（從 `config/config.yml.bak` 複製）

**啟動：**
```bash
cp config/config.yml.bak config/config.yml
# 編輯 config.yml，填入 db / psl / websocket 設定
go build -tags dev ./...
```

#### 模組說明

| 套件 | 職責 |
|---|---|
| `pkg/psl` | PSL API 介接層，所有幣流分析 RPC 的統一入口 |
| `psl/explain_transaction` | 解析單筆交易的資金流向（Effects / Fees） |
| `psl/list_transactions` | 依地址查詢歷史幣流記錄 |
| `psl/get_balance` | 查詢指定地址在特定鏈上的即時餘額 |
| `psl/entity` | 產生標準 PSL Entity ID（`GenerateEntity`） |
| `psl/auth` | PSL API 認證管理（支援 `sha512` / `basic` / `none`） |
| `pkg/blockchain` | 多鏈對應層，將內部 `BlockchainType` 轉換為 PSL network string |
| `pkg/websocket` | 透過 WebSocket 即時推送幣流狀態更新至前端 |
| `pkg/database` | PostgreSQL 操作層 |

#### PSL Entity ID 格式

| 類型 | 格式 | 範例 |
|---|---|---|
| 地址 | `{network}/addr/{address}` | `ethereum.mainnet/addr/0xABC...` |
| ERC-20 Token | `{network}/token/erc20/{contract}` | `ethereum.mainnet/token/erc20/0xUSDT...` |
| 原生代幣 | `{network}/token/native` | `ethereum.mainnet/token/native` |
| 交易 | `{network}/txn/{tx_hash}` | `ethereum.mainnet/txn/0xTXHASH...` |

#### PSL `explain_transaction` 回傳結構

```
Effects
  └── AdjustBalance  → 地址餘額變動（sender / receiver / amount）
  └── NftTransfer    → NFT 轉移記錄

Fees
  └── Exact   → 精確手續費金額
  └── Range   → 手續費範圍（min / max）
```

#### 支援區塊鏈（PSL 網路對應）

| 鏈 | PSL Network String |
|---|---|
| Ethereum | `ethereum.mainnet` / `ethereum.sepolia` |
| Bitcoin | `bitcoin.mainnet` / `bitcoin.testnet` |
| Solana | `solana.mainnet` / `solana.devnet` |
| Flow | `flow.mainnet` / `flow.testnet` |
| Conflux | `conflux.mainnet` |
| Band | `band.laozi-mainnet` |
| Fantom | `fantom.opera` |
| BSOS | `bsos.mainnet` |

#### 排程工作（`job` 設定）

| 工作 | 預設間隔 | 說明 |
|---|---|---|
| `sync.transaction.state` | 每 1 分鐘 | 同步未完成交易的確認狀態 |
| `subscribe.wallet` | 每 5 分鐘 | 自動補訂錢包地址訂閱 |

#### 設定重點（`config.yml.bak`）

```yaml
psl:
  auth.method: "sha512"   # basic | sha512 | none
  url: "https://walletos.pinestreetlabs.com/rpc/"
  password: "****"
  apikey: "****"
  apisecret: "****"

websocket:
  app.prefix: "aegisvault_dev"
  url: "ws://localhost:8081/ws"
  networks: "BITCOIN_TESTNET,ETHEREUM_MAINNET,ETHEREUM_SEPOLIA"
  ping.interval: "60s"
  reconnect:
    retry: 5
    delay: "1s"

job:
  sync.transaction.state.minutes: 1
  subscribe.wallet.minutes: 5
```

#### 資料庫 Schema

| 資料表 | 說明 |
|---|---|
| `ws_raw_transaction` | 原始交易記錄（full_tx_id 唯一，含 state / processed 旗標） |
| `ws_token_subscription` | Token 訂閱記錄（network + address 唯一） |

### 3.3 crypto-event-ws（WebSocket 廣播）

**職責：** 消費 Kafka 事件，透過 WebSocket 即時推播交易通知至前端（aegis-vault-app）及下游訂閱者。

**設定檔：** `config/config.yml`（從 `config/config.yml.bak` 複製）

**啟動：**
```bash
cp config/config.yml.bak config/config.yml
go build -tags dev ./...
```

#### 設定重點（`config.yml.bak`）

```yaml
http:
  listen:
    addr: "127.0.0.1"
    port: 8081

kafka:
  broker.urls: "localhost:9092"
  consumer.group: "ce-ws-grp"
```

### 3.4 transaction-listener（私有鏈 RPC 輪詢）

**職責：** 針對不支援事件訂閱的私有鏈（如 Hyperledger Besu），以 RPC 輪詢方式追蹤區塊交易，並透過 Webhook 通知下游服務。

**架構：** 雙 App 設計
- `apps/api` — HTTP API 服務（管理訂閱、API Key）
- `apps/listener` — 背景監聽服務（區塊輪詢、通知推播）

兩個 App 共用同一個 PostgreSQL 資料庫（`tx-listener-db`）。

**啟動：**
```bash
# API 服務
cd apps/api
cp config/config.yml.bak config/config.yml
go build -tags dev ./...

# Listener 服務
cd apps/listener
cp config/config.yml.bak config/config.yml
go build -tags dev ./...
```

#### Listener 設定重點（`apps/listener/config.yml.bak`）

```yaml
blockchain.module:
  type: "eth"
  network: "hyperledger.besu"
  base.url: "http://xxx.xxx.xx"   # 私有鏈 RPC endpoint
  native.symbol: "ETH"
  native.decimals: 18
  fetch.interval: 15    # 秒，抓取區塊間隔
  sync.interval: 3      # 秒，狀態同步間隔
  batch.size: 10        # 每批處理區塊數
  safe.confirms: 20     # 安全確認數

cron.time:
  push.notification: "* * * * *"   # cron，推送通知
  sync.notification: "* * * * *"   # cron，同步通知狀態
```

#### 資料庫 Schema

| 資料表 | 說明 |
|---|---|
| `block_numbers` | 各鏈已同步的最新區塊號 |
| `block_txs` | 已發現的區塊交易（from / to / amount / notified_at） |
| `token_infos` | Token 合約資訊快取（symbol / decimals） |
| `transfers` | 解析後的轉帳記錄（支援 ERC-20 / NFT / native） |
| `api_keys` | 客戶端 API Key（含 Webhook URL） |
| `subscriptions` | 地址訂閱記錄（含 confirmations 設定） |
| `notifications` | 待推播通知佇列 |
| `push_logs` | 推播成功 / 失敗紀錄 |
| `skipped_blocks` | 跳過的異常區塊記錄 |

## 四、基礎設施依賴

| 依賴 | 用途 | 涉及服務 |
|---|---|---|
| **PostgreSQL** | 持久化訂閱、事件、幣流記錄 | 所有四個服務各自獨立 DB |
| **Kafka** | 事件佇列（訂閱指令 / 交易事件） | crypto-event-subscriber ↔ aegisvault-transaction, crypto-event-ws |
| **CryptoAPIs** | 外部公鏈事件訂閱（需 API Key） | crypto-event-subscriber |
| **PSL（Pine Street Labs）** | 交易語義解析 RPC | aegisvault-transaction |
| **私有鏈 RPC** | Hyperledger Besu 等私有鏈節點 | transaction-listener |

## 五、部署注意事項

1. **資料庫遷移：** 每個服務均包含 `sqls/` 資料夾，需在啟動前依序執行 `*.up.sql` 遷移腳本。
2. **Kafka Topic 前綴：** `crypto-event-subscriber` 的 `kafka.topic.prefix` 需與消費端（`aegisvault-transaction`、`crypto-event-ws`）一致。
3. **PSL API 憑證：** `aegisvault-transaction` 依賴外部 PSL 服務，需申請 `apikey` / `apisecret`，並設定 `auth.method: sha512`。
4. **CryptoAPIs Callback URL：** `crypto-event-subscriber` 需提供可被外部回調的 HTTPS endpoint，用於接收鏈上事件通知。
5. **私有鏈輪詢：** `transaction-listener` 僅適用於無事件訂閱支援的鏈（如 Hyperledger Besu），公鏈請使用 `crypto-event-subscriber`。
6. **WebSocket 認證：** `aegisvault-transaction` 連接 `crypto-event-ws` 時需設定 `websocket.apikey` / `websocket.apisecret`。
